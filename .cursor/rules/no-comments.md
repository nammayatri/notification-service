## Do not add comments.

Default: no comments. Let well-named identifiers carry the *what*. Only add a comment when the **why** is genuinely non-obvious — a hidden constraint, a subtle invariant, a workaround for a specific bug whose absence would mislead a future reader. Never narrate *what* the code does, never reference the current task / PR / caller / ticket number.

If a comment would be removed without confusing anyone, don't write it.
